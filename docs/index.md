## About

PwnzzAI Shop represents a **hands-on learning platform** purposefully built for instruction on AI technology protection. This intentionally insecure Flask-based web application showcases an extensive array of AI security weaknesses via an immersive pizza shop experience.

**Founding partner**: [OWASP AI Exchange](https://owaspai.org/)

### Scope and Learning Goals

PwnzzAI is designed as a comprehensive, evolving learning platform aligned with the **[AI Exchange risk taxonomy](https://owaspai.org/docs/ai_security_overview/#threats-overview)**. Its scope expands progressively in step with OWASP AI Exchange security analysis frameworks and defensive guidance, ensuring long-term relevance as AI security practices mature.

The project currently incorporates the OWASP Top 10 for LLMs, with an architecture intentionally mapped to AI Exchange risk classifications. Over time, PwnzzAI aims to mature into a structured learning ecosystem that supports:

- AI protection and defense training, including self-directed learning paths
- Hands-on security training programs with practical, scenario-driven exercises
- Team-based learning initiatives for security engineers and practitioners
- End-to-end AI security education, spanning design, development, deployment, and operations

## Features

- **Pizza Service**: Browse appetizing pizzas, explore customer feedback, contribute comments, and place virtual orders!
- **Extensive Weakness Showcases**: Active demonstrations spanning OWASP LLM Top 10 and AI Exchange threat categories.
- **Learning-Focused Design**: Discover exploitation methods and defensive approaches via transparent descriptions.

## AI Security Coverage

This platform showcases an extensive spectrum of AI protection challenges, aligned with the **[OWASP AI Exchange comprehensive threat taxonomy](https://owaspai.org/docs/ai_security_overview/)**.

### Implemented Vulnerabilities

According to [OWASP AI Exchange threats](https://owaspai.org/docs/ai_security_overview/) and [OWASP Top 10 for LLM Applications 2025](https://genai.owasp.org/resource/owasp-top-10-for-llm-applications-2025/):

1. **Prompt Injection** - Direct and indirect prompt injection attacks
2. **Data Disclosure** - Extraction of sensitive information through model outputs
3. **Supply Chain Vulnerabilities** - Third-party model and plugin security risks
4. **Data and Model Poisoning** - Malicious training data manipulation
5. **Improper Output Handling** - Unvalidated LLM outputs leading to injection attacks
6. **Excessive Agency** - Over-privileged LLM operations without appropriate constraints
7. **System Prompt Leakage** - Extraction of secret information from system prompts
8. **Vector and Embedding Weakness** - Unauthorized access to embeddings in RAG systems
9. **Misinformation** - Hallucinations and harmful misinformation generation
10. **Unbounded Consumption** - Resource exhaustion attacks and denial of service
11. **Model Theft** - Model extraction and intellectual property theft
